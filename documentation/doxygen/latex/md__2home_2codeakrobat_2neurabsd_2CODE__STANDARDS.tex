This document defines the coding standards for the Neura\+BSD project. To ensure seamless compatibility with our upstream base while maintaining a distinct identity, we combine {\bfseries{Open\+BSD KNF}} with the {\bfseries{Neura\+BSD Hybrid Style}}.

For all C code (Kernel, Base System, and core utilities like {\bfseries{Daemon\+Path}}), Neura\+BSD strictly adopts the official {\bfseries{Open\+BSD KNF (Kernel Normal Form)}}.


\begin{DoxyItemize}
\item {\bfseries{Reference\+:}} \href{https://man.openbsd.org/style.9}{\texttt{Open\+BSD style(9) manual page}}
\item {\bfseries{Indentation\+:}} Hard Tabs (8 characters wide).
\item {\bfseries{Braces\+:}} KNF convention (Braces on the same line for {\ttfamily if}, {\ttfamily for}, {\ttfamily while}).
\item {\bfseries{Naming\+:}} {\ttfamily snake\+\_\+case} for all C functions and variables.
\end{DoxyItemize}\hypertarget{md__2home_2codeakrobat_2neurabsd_2CODE__STANDARDS_autotoc_md29}{}\doxysubsection{\texorpdfstring{2. Neura\+BSD Python Style (Hybrid)}{2. Neura\+BSD Python Style (Hybrid)}}\label{md__2home_2codeakrobat_2neurabsd_2CODE__STANDARDS_autotoc_md29}

\item Classes\+: {\ttfamily Pascal\+Case}.

\item {\bfseries{Line Length\+:}} Maximum 80 characters.
\item {\bfseries{Documentation\+:}} Concise docstrings in {\ttfamily mdoc(7)} style.
